# Path Copy

![Path Copy アイコン](assets/path-copy-icon.png)

**VS Code を離れずに、必要なファイルパスや Git リポジトリ URL をコピーできます。**

Path Copy は、エクスプローラーとエディタータブに IntelliJ 風のコピー選択ダイアログを追加します。絶対パス・ワークスペースパス・リポジトリパス・ファイル名・リポジトリ URL の実際の値を確認してからコピーできます。

[English](README.md) · [Issue を報告](https://github.com/noritaka1166/vscode-path-copy/issues) · [ソースコード](https://github.com/noritaka1166/vscode-path-copy)

## Path Copy を使う理由

- **コピー前に値を確認できる** — 各候補の右側に実際の値が表示されるため、目的に合う参照を確実に選べます。
- **コードの場所をすばやく共有できる** — ファイルやフォルダーの右クリックメニューから、パスやリポジトリ URL を直接コピーできます。
- **実際のプロジェクト構成に対応** — マルチルートワークスペースでも、選択した項目のワークスペースフォルダーと最も近い Git リポジトリを自動で判定します。
- **アカウント・ネットワーク接続は不要** — パスと Git リモートはローカルで読み取り、テレメトリーも送信しません。

## 使い始める

1. VS Code Marketplace から **Path Copy** をインストールします。
2. エクスプローラー内のファイル／フォルダー、またはエディタータブを右クリックします。
3. **パスまたはリポジトリ URL をコピー…** を選びます。
4. 必要な値を選択すると、すぐにクリップボードへコピーされます。

長い値はダイアログ上で省略表示されます。候補の右端にある情報アイコンへカーソルを合わせると全量を確認でき、アイコンのクリックでもコピーできます。

## コピーできるもの

| コピー項目 | 使う場面 |
| --- | --- |
| 絶対パス | ターミナルコマンド、スクリプト、OS のファイル選択ダイアログでローカルパスが必要なとき |
| ファイル名 | 選択したファイルまたはフォルダーの名前だけが必要なとき |
| コンテンツルートからのパス | 現在の VS Code ワークスペースフォルダーからの相対パスが必要なとき |
| リポジトリルートからのパス | ドキュメント・Issue・レビューで安定したプロジェクト相対パスを共有したいとき |
| リポジトリ URL | 選択項目を含む Git リポジトリへのリンクを共有したいとき |

Git 関連の項目では、選択した項目を含む最も近いリポジトリを使います。マルチルートワークスペースでは、選択した項目が属するワークスペースフォルダーを基準にします。

## リポジトリ URL の形式

既定では、`git@github.com:owner/repository.git` のような SSH リモートを、共有しやすい `https://github.com/owner/repository` のようなブラウザー URL に変換します。

Git に設定されたリモート URL をそのままコピーするには、次を設定してください。

```json
{
  "pathCopy.remoteUrlFormat": "preserve"
}
```

`origin` 以外のリモートを使うには、次を設定します。

```json
{
  "pathCopy.remoteName": "upstream"
}
```

## 対応言語

VS Code の表示言語に自動追従します。既定の英語に加え、日本語・中国語（簡体字）・韓国語・フランス語・ドイツ語・スペイン語・ポルトガル語（ブラジル）を収録しています。

## 動作要件

- VS Code 1.85 以降
- リポジトリ相対パスとリポジトリ URL を使う場合のみ Git が必要です
